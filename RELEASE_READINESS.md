# Release Readiness

Дата: 2026-04-19

## Что готово

- Реализован базовый API-контур (`ApiClient`, ошибки, конфиг).
- Введен слой репозиториев с переключением `mock/remote`.
- Добавлены DTO/маппинги доменных моделей.
- Реализовано разграничение прав для админ-панели:
  - скрытие пункта админки для non-admin,
  - guard доступа на экране админки.
- Реализован frontend flow показаний счетчиков:
  - выбор фото (камера/галерея),
  - OCR-вызов,
  - ручная правка результата,
  - отправка показаний.
- Убран основной набор "мертвых" кнопок/ссылок.
- Добавлены smoke/access/OCR тесты, тестовый прогон зеленый.

## Что остается техдолгом

- Remote endpoints пока предполагаются контрактно (`/meters/ocr`, `/meters/readings`, `/user/me` и др.) — нужны финальные backend-контракты.
- В ряде экранов используются fallback mock-данные при ошибках сети; для production стоит сделать более строгие состояния offline/error.
- Нет полноценного e2e UI-интеграционного теста с реальным backend окружением.
- Отсутствует централизованная телеметрия/логирование ошибок в внешнюю систему.
- Не реализованы бизнес-секции админки (кроме заявок) — помечены как disabled.

## Минимальный pre-release чеклист

- [ ] Проверить `API_BASE_URL` и `DATA_SOURCE=remote` для release-сборки.
- [ ] Прогнать `flutter test` в CI и локально.
- [ ] Проверить вход под `admin` и `resident`:
  - [ ] visibility админ-пункта,
  - [ ] отказ в доступе для non-admin при прямом переходе.
- [ ] Прогнать OCR-сценарий в ручную:
  - [ ] выбор фото,
  - [ ] распознавание,
  - [ ] ручная правка,
  - [ ] отправка.
- [ ] Пройти smoke-навигацию по основным разделам (Home/Requests/Payments/Meters/Admin).
- [ ] Проверить UX ошибок:
  - [ ] OCR error + retry,
  - [ ] submit error + retry.
- [ ] Сверить, что отключенные админ-блоки не кликаются и явно отмечены.

## Рекомендация перед продом

Если релиз в production планируется сразу после интеграции backend, стоит добавить короткий этап hardening:

1. Вынести API-контракты в отдельные typed DTO под финальную схему backend.
2. Добавить явные error states на ключевых экранах (не только snackbars).
3. Подключить crash/error monitoring.
